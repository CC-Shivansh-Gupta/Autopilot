# Chapter 8 — State Estimation

This chapter implements the three-stage state estimation pipeline described in Beard & McLain Chapter 8. The first stage is a simple low-pass filter that recovers airspeed, angle of attack, and sideslip from the pressure sensor measurements (Section 8.4). The second stage is an attitude Extended Kalman Filter (EKF) with a two-element state vector [phi; theta]; it propagates using kinematic equations driven by gyro inputs and performs measurement updates from the accelerometers (Section 8.9, Eqs 8.19–8.29). The third stage is a GPS EKF with a seven-element state vector [pn; pe; Vg; chi; wn; we; psi] that fuses GPS position, ground speed, and course angle measurements to estimate horizontal navigation states and wind (Section 8.10, Eqs 8.34–8.50).

All filter states are stored in MATLAB `persistent` variables so that `estimate_states.m` can be called once per timestep inside the simulation loop without reinitialising. Students must tune the process and measurement noise covariance matrices `Q_att`, `R_att`, `Q_gps`, and `R_gps` (added to `F4_params.m`) to achieve acceptable estimation accuracy. The test script `Section8_Estimation.m` runs a 60-second closed-loop simulation and plots true versus estimated values for roll angle, pitch angle, north/east position, altitude, and airspeed.
