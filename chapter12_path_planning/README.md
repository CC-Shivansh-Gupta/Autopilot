# Chapter 12 — Path Planning (RRT)

This chapter implements Rapidly-exploring Random Tree (RRT) path planning in an urban obstacle environment, following Beard & McLain Chapter 12. Two variants are provided in `rrt.m`: Algorithm 11 is the straight-line RRT, which grows the tree by stepping a fixed distance (`P.rrt_step`) toward random samples and checking straight-line segments for building collisions; Algorithm 12 is the Dubins RRT, which replaces each straight step with a minimum-length Dubins path arc (calling `dubins_parameters.m` from Chapter 11) so that all waypoints are reachable by the minimum-radius-constrained vehicle. After the tree reaches the goal, a path-smoothing pass removes unnecessary intermediate nodes whose direct Dubins connection is collision-free.

The obstacle map is a struct containing a list of buildings described as axis-aligned boxes with position (pn, pe), width, and height in feet. Students must implement the collision-check sub-function inside `rrt.m` (axis-aligned bounding-box test for straight-line segments, discretised arc test for Dubins segments) and the nearest-node and steering logic. Two parameters are needed in `F4_params.m`: `P.R_min` (minimum turn radius, shared with Chapter 11) and `P.rrt_step` (step size, typically 1–2× the turn radius). The test script `Section12_PathPlanning.m` generates a random five-building map and plots both planned paths in a 3-D figure.
