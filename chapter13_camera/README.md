# Chapter 13 — Gimbal Model and Camera

This chapter implements a stabilised gimbal pointing model and pinhole camera projection for target tracking, following Beard & McLain Chapter 13. The main function `gimbal_model.m` takes the full 12-state MAV vector and a target position in the NED frame and returns the required gimbal azimuth and elevation angles plus the projected image-plane coordinates of the target. The rotation chain is: NED to body frame via the standard rotation matrix R_bv (from phi, theta, psi), then body to gimbal frame via sequential rotations about the body-z (azimuth) and gimbal-y (elevation) axes, and finally a pinhole projection onto the image plane using focal length `P.f_cam` in pixels. The azimuth and elevation angles are computed by expressing the MAV-to-target vector in the body frame and taking arctangents.

Two camera parameters must be added to `F4_params.m`: `P.f_cam` (focal length in pixels, e.g. 500 px) and `P.img_width` / `P.img_height` (sensor size in pixels, e.g. 640x480). The test script `Section13_Camera.m` places the MAV on a circular orbit at 200 ft altitude around a stationary ground target at [500, 500, 0] ft and simulates one full orbit, plotting the ground track, the gimbal azimuth and elevation angles over time, and the image-plane trajectory of the target — which should form a closed loop for a perfect circular orbit.
