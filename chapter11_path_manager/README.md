# Chapter 11 — Path Manager

This chapter implements the path manager layer that sits above the path follower: given a list of waypoints, it decides which path primitive (line segment, fillet, or Dubins arc) to hand to `path_follower.m` at each instant, and switches to the next segment when the MAV crosses the appropriate half-plane. Three manager algorithms are implemented following Beard & McLain Chapter 11: Algorithm 7 (straight-line segments, immediate half-plane switching), Algorithm 8 (fillet segments, which insert small circular arcs at each corner to avoid abrupt heading changes), and Algorithm 9 (Dubins segments, which chain the minimum-length Dubins path between consecutive waypoint headings). The Dubins geometry — start circle, straight segment, end circle — is computed by `dubins_parameters.m` (Algorithm 10), which evaluates all four Dubins cases (RSR, RSL, LSR, LSL) and returns the shortest one.

The minimum turn radius `P.R_min` must be added to `F4_params.m`; for the F-4 at trim airspeed, a value of roughly 300 ft is appropriate. The test script `Section11_PathManager.m` flies a four-waypoint rectangular circuit at 100 ft altitude using all three manager types and overlays their ground tracks on one figure, making it easy to compare the smoothness of each approach.
